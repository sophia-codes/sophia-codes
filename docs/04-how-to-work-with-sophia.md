# how to work with sophia

> clear ask. clear lane. clear outcome.

sophia works best through structured collaboration.
not because the process needs to be heavy, but because good work usually gets better when the shape of the collaboration is clear.

---

## the simplest model

start with:
- one project or repo
- one current objective
- one working mode
- one desired outcome

that is enough to begin well.

---

## working modes

### strategic advisor
use sophia when you need:
- architecture judgment
- execution planning
- roadmap refinement
- structured thinking before implementation expands

### execution partner
use sophia when you need:
- plans turned into concrete work
- issues, docs, and prs shaped clearly
- continuity across an active stream of work

### review and refinement layer
use sophia when you need:
- implementation review
- maintainability review
- architecture drift detection
- docs or repo cleanup

### embedded technical co-builder
use sophia when you want her inside the delivery rhythm:
- shaping issues
- contributing code/docs
- reviewing plans
- preserving clarity across the work over time

---

## the best way to begin

bring:
- one problem
- one repo or project area
- one current bottleneck
- one desired outcome

then let sophia help define:
- the cleanest next move
- the right shape of the work
- the level of effort worth taking on first

---

## what good collaboration looks like

healthy collaboration with sophia usually has this rhythm:

1. frame the objective
2. provide bounded context
3. get a structured output
4. review intentionally
5. continue only where there is signal

this keeps the work calm, visible, and useful.

---

## example asks

- review this architecture for hidden coupling and structural risk
- turn this feature into the cleanest execution plan
- audit this phase against current repo reality
- review this pr for maintainability and boundary drift
- refine this document into something clearer and more operational

---

## what to expect from sophia

done well, working with sophia should produce:
- better framing
- clearer decisions
- stronger technical artifacts
- more coherent next steps
- less drift between intention and implementation

---

## closing line

> the best sophia collaborations are not unbounded. they are well-shaped.
